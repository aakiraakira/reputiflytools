Protocol C Standard Operating Procedure

E-Commerce Data Ingestion Protocol

This document serves as the authoritative guide for submitting product data to the web platform. The system utilizes an Automated Ingestion Engine. To ensure zero errors during the synchronization of prices, images, and inventory, adhering to the formatting standards below is mandatory.

! Critical Compliance Rules

Rule #1: Numeric Integrity

The system rejects currency symbols. Input raw integers or decimals only.

INCORRECT $25.00
CORRECT 25.00
Rule #2: Asset Linking

Images must be provided via Dropbox Share Links. Direct file pastes are incompatible with the database.

✅ https://dropbox.com/scl/fi/xyz...

Data Field Specifications

The ingestion matrix consists of 9 columns.

Col Header Required? Specification
A Product Name YES Public-facing title. e.g. "Fingerprint Lock Model X"
B Type YES Select "Simple" or "Variable" from the dropdown menu.
C Base Price YES Standard retail price. Numbers only.
D SKU OPTIONAL Stock Keeping Unit. Leave blank to auto-generate.
E Variations CONDITIONAL Required only for "Variable" type. e.g. "Size: Small, Large"
F Specific Prices OPTIONAL Overrides for specific sizes. e.g. "Large=200"
G Main Image YES Primary thumbnail (Dropbox Link).
H Gallery Images OPTIONAL Additional photos, comma-separated.
I Description OPTIONAL Full product details and specifications.
Scenario 1

Simple Product Configuration

Definition: A singular item with no selectable options (e.g., A Standard Door Lock).

Visual Guide: Input Matrix
A: Name B: Type C: Base Price D: SKU E: Variations F: Specific Prices G: Main Image H: Gallery Images I: Description
My Product Simple 50 SKU-001 dropbox... Details...
Required Inputs:
  • Column B (Type): Must be set to "Simple".
  • Column C (Base Price): Enter the single price.
  • Column G (Main Image): Provide one Dropbox link.
Prohibited Inputs:
  • Column E (Variations): Must be completely blank.
  • Column F (Specific Prices): Must be completely blank.
Scenario 2

Variable Product (Standard)

Definition: An item with options (e.g., Size, Color) where all options share the same price.

Visual Guide: Input Matrix
A: Name B: Type C: Base Price D: SKU E: Variations F: Specific Prices G: Main Image
T-Shirt Variable 25 SHIRT-01 Size: S, M, L dropbox...
Column E Syntax Protocol

You must specify the Category Name followed by a colon, then comma-separated options.

✅ Size: Small, Medium, Large ❌ Small, Medium, Large (Error: Missing 'Size:' prefix)
Scenario 3

Complex Configuration (Price Overrides)

Definition: Variable products where specific options have different costs (e.g., XL costs +$10) and require additional gallery photos.

Visual Guide: Input Matrix
C: Base Price E: Variations F: Specific Prices G: Main Image H: Gallery Images
100 Size: S, L, XL L=120, XL=140 dropbox... dropbox..., dropbox...
Price Override Logic
  • 1. Base Price (100): This applies to any option NOT listed in Column F (e.g., Small).
  • 2. L=120: Directs the system to price "Large" at 120.
  • 3. XL=140: Directs the system to price "Extra Large" at 140.
Gallery Protocols
  • 1. Column G (Main): Defines the catalog thumbnail.
  • 2. Column H (Gallery): Defines supplementary views. Separate multiple links with a comma.
