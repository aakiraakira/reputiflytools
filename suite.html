REPUTIFLY SOFTWARE SUITE

The complete technical and operational documentation for the end-to-end proprietary enterprise ecosystem.

Executive Summary

The Reputifly Software Suite is a proprietary, end-to-end enterprise ecosystem designed to systemize, automate, and scale web design and SEO agency operations.

It fundamentally shifts the agency model from a labor-intensive service to a highly automated, SaaS-like fulfillment machine. By bypassing traditional, bloated platforms (like WordPress) and leveraging a serverless stack (GitHub REST API for versioning/hosting, Firebase for real-time state, RBAC, and routing), the suite protects profit margins, eliminates scope creep, and creates a plug-and-play environment for new developers.

Strict Operational Lifecycle

Lead Capture (Funnel) Scoping (Scope) AI Development (Forge) Client Revisions (Prism) Deployment (Foundry) Routing (API) Management (Nexus)

MODULE 01

Reputifly Funnel

The Automated Top-of-Funnel (TOFU) Lead Engine

View Live Demo

Core Function

The Funnel acts as an automated pre-sales engineer. It captures inbound leads and forces them to provide highly structured, AI-ready data. When the resulting free coded sample is paired with our automated Scope PDF, it delivers an overwhelming amount of upfront value to the client—drastically increasing close rates while entirely eliminating manual labor.

Deep Technical Functionality

Gamified Data Extraction (The Strength Meter)

The intake form utilizes a dynamic, algorithm-driven "Detail Strength Meter." By psychologically gamifying the input—shifting from a red "Weak" warning to a green "Excellent" validation based on alphanumeric character density—it tricks the lead into writing comprehensive, high-quality prompts about their target audience and unique selling propositions.

Client-Side Asset Pre-Processing

To protect server costs and prevent payload crashes from massive files, the Funnel intercepts uploaded client logos entirely within the browser. It draws the raw image to an invisible HTML5 <canvas>, calculates optimal aspect ratios (capping at a max 1200px dimension), and aggressively compresses it to a lightweight JPEG (strictly under 3MB) before any server interaction occurs.

Brand Constraint Enforcement

If a lead lacks a logo, the UI blocks progression until they select from predefined, Tailwind-compliant color palettes (e.g., Ocean, Sunset, Rose) and premium typography pairings (e.g., Space Grotesk, Inter). This entirely bypasses the traditional "I don't know what I want" phase of client onboarding.

Dual-Pipeline Routing Strategy

1. The CRM Payload: It fires a clean, sanitized JSON payload to the standard backend API endpoint to log the lead and trigger internal webhook notifications.

2. The AI Bypass: Simultaneously, it concatenates the extracted brand colors, fonts, industry, and the "strength meter" description into a FormData payload tagged with a strict [MANUAL_MODE_BYPASS] instruction. This hits the internal AI generation endpoint, silently spinning up a coded Tailwind sample while the sales team is still reading the initial notification.

Instant Conversion Trap

Upon submission, the UI immediately redirects the prospect to a success screen and, after a 1.5-second delay, automatically triggers a pre-filled WhatsApp link (wa.me). This instantly transitions a cold web lead into a direct, high-intent, human-to-human chat conversation.

MODULE 02

Reputifly Scope

The Architecture, Estimation & Contract Builder

Core Function

Scope is the ultimate antidote to agency unprofitability. It is a rigid proposal builder that entirely eliminates inaccurate estimation. By formally LOCKING the sitemap and standardizing deliverables upfront, it establishes exact structural boundaries before a single line of code is written, effectively killing scope creep before it starts.

Deep Technical Functionality

Visual Sitemap Locking

The system provides an interactive, drag-and-drop-style UI to dynamically add, name, and categorize pages. This visual blueprint forces the agency and the client to agree on—and permanently lock—the exact structural limits of the website during the sales process.

AI-Assisted Contextualization

Developers utilize an integrated LLM connection to automatically generate professional, detailed architectural descriptions for specific sections within the sitemap. The AI parses the page intent and maps out the required logic (e.g., parsing an "About" page into "Hero Section, 3-Column Team Grid, Mission Statement, CTA"), ensuring absolute clarity for the assigned developer.

Algorithmic Feature Flagging & Estimation

By checking off specific technical requirements (e.g., CMS integration, advanced scroll animations, third-party API connections), the system mathematically calculates the objective timeline and pricing. This completely removes emotion and guesswork from the sales and quoting process.

Direct Database Injection

When a scope is finalized, it doesn't just generate a static PDF for the client. The deeply structured JSON data is pushed directly into the Firebase projects collection. This instantly establishes the live project environment for the development team, pre-configuring the exact pages, routing, and features they need to build in Forge.

MODULE 03

Reputifly Forge

The AI-Powered UI Component Factory

Core Function

Forge is the development accelerator. It allows internal teams (even junior-level hires) to bypass manual CSS/HTML coding by acting as an on-demand, AI-driven Tailwind component factory.

Deep Technical Functionality

Prompt-to-Code Pipeline

Interfaces directly with backend Large Language Models (LLMs). A developer types a natural language prompt, and Forge returns clean, responsive, production-ready HTML and Tailwind CSS code tailored to the project's Scope.

Intelligent JSON Brand Extraction

Integrates color-thief and Cropper.js. Developers can upload a client's reference image or logo; the system then calculates and extracts the dominant color palette. Crucially, these brand styles are instantly parsed into strict JSON token structures. Developers inject these precise JSON tokens into the AI prompts, guaranteeing the LLM applies perfectly accurate HEX/RGB values across every generated component.

Split-Pane Iteration Environment

The UI features a highly efficient split-pane design. It renders the generated code in a live iframe on one side while exposing the raw HTML in a syntax-highlighted editor on the other.

Contextual AI Refinement

If a generated section is slightly off, the developer does not need to start over or manually edit the DOM. They can prompt the AI to iterate specifically on the existing output (e.g., "Make the primary buttons rounded and add a purple hover glow"), drastically reducing development hours and debugging time.

MODULE 04

Reputifly Prism

The Automated Client Revision Pipeline (ACRP)

Explore Prism

Core Function

Prism is the ultimate scope-creep killer and the bridge between client feedback and developer execution. It mathematically constrains visual revisions and translates vague client desires into actionable, AI-ready code instructions.

Deep Technical Functionality

Visual Anchoring (The Feedback Layer)

Clients view the live staging link and interact directly with the DOM. They drop visual pins onto specific elements to attach instructions or new image assets. The UI features a "cyber-loader" initialization screen and modal system to process this feedback without breaking the underlying iframe.

Google Docs ETL Protocol (The Ingestion Engine)

Revisions are processed through a strictly enforced "Traffic Light" layout in Google Docs.

  • Red Zone: Text-based instructions and logic changes.
  • Green Zone: Production-ready image/video assets.

Prism acts as an Extract, Transform, Load (ETL) engine. It scrapes the published web version of the Google Doc programmatically (bypassing auth walls) and reads the semantic hierarchy (H1 tags = Pages, H2 tags = Sections) to map feedback contextually.

Cursor AI Execution (The Output Protocol)

Prism compiles the parsed DOM pins and the scraped Google Docs instructions into a highly structured markdown file known as a "Cursor Pack" (cursor_instructions.md). The developer drags this pack into the Cursor IDE. The AI coding agent reads the pack, mapping the logical instructions (Layer 1) directly to the physical assets in the codebase (Layer 2), executing the revisions autonomously across index.html and stylesheets.

MODULE 05

Reputifly Foundry

The Serverless Edge Deployment Engine

Explore Foundry

Core Function

Foundry completely eliminates the need to juggle dozens of fragile SiteGround or cPanel logins. It replaces outdated web hosting paradigms with a bespoke serverless engine that bridges the gap between local development and live edge networks, utilizing GitHub as a headless CMS and version control system.

Deep Technical Functionality

Smart Asset Parsing via JSZip

The drag-and-drop UI processes entire project folders locally within the browser. It intelligently flattens complex directories, hunts for the root index.html file, and queues all static assets (images, scripts, styles) into memory without requiring heavy server-side processing or temporary backend storage.

Transparent Version History Control

It maps directly to the GitHub REST API. Deployments are pushed to specific repositories under distinct branches (e.g., base, v1, v2). This ensures an immutable, easily reversible history of the codebase, allowing clients and developers to seamlessly toggle and compare complete visual versions of the site side-by-side.

REST API Chunking & Pushing

Converts local files into base64 encoded payloads and executes multi-file PUT requests to the GitHub API. This effectively pushes live code updates to the edge network in seconds without the overhead of traditional deployment pipelines.

Audit Logging & Accountability

Every critical deployment action, branch creation, or file overwrite is intercepted and forwarded to a secure Telegram webhook, providing the agency owner with a real-time, transparent audit trail of all developer activity.

MODULE 06

Reputifly Foundational API

The Serverless Nervous System & Routing Hub

Core Function

Operating via Firebase Cloud Functions V2, this API completely decouples the frontend code from backend logic. It replaces bloated, insecure CMS plugins (like WordPress Contact Form 7) with a centralized, proprietary routing system.

Deep Technical Functionality

Universal Payload Routing

Static HTML builds deployed via Foundry send lightweight POST requests directly to this API. The API sanitizes the data, injects it securely into Firestore, and triggers multi-channel notifications (Resend for Email, Telegram webhooks). This ensures zero leads are lost and deployed sites maintain perfect Core Web Vitals (99+ scores) by offloading backend bloat.

Strict Role-Based Access Control (RBAC)

The API is protected by sophisticated JWT decoding. The checkAccess function verifies the SUPER_ADMIN_UID for overarching control. For standard users, it cross-references their authenticated email against the assignedEmployees array in Firestore, ensuring employees can only deploy or modify projects they are explicitly assigned to.

Automated Infrastructure Maintenance

Utilizes Firebase onSchedule cron jobs to autonomously manage cloud storage. It sweeps the design_packs/ storage buckets daily, automatically deleting AI-generated assets or bloated project files older than 7 days, aggressively optimizing server costs and preventing storage limits from being breached.

Global State Management

Acts as the secure gatekeeper for the entire suite—handling the creation of new client accounts, pausing access, setting custom API rate limits per client, and forcefully triggering agency-wide metric reports.

MODULE 07

Reputifly Nexus

The Command Center & Client Staging Portal

Explore Nexus Features

Core Function

Nexus is infinitely more than a dashboard—it is the comprehensive operational brain of Reputifly. It aggregates the entire suite into a single, high-converting interface. It centralizes project financials, orchestrates seamless developer-to-client handoffs, manages the overarching workflow, and provides the mathematically constrained staging environment for client approvals.

Deep Technical Functionality

The Agency Dashboard (Internal)

A bird's-eye command center that tracks inbound leads from the Foundational API, monitors deployment statuses from Foundry, and manages repository branching. The UI utilizes advanced CSS shimmer skeletons and dynamic "AI Analysis" loading states to maintain a premium feel while fetching heavy data.

The Client Portal (External)

A highly polished, secure staging environment. Clients bypass messy email links with zip files and instead log into a portal where they can navigate a multi-page selector, read developer release notes (rendered dynamically via marked.js), and interact with strategic agency cross-sells (e.g., SEO/Ads packages) embedded directly into the UI.

The Visual Workspace

A wildly sophisticated client-side WYSIWYG editor. It loads the raw HTML into an iframe, injects custom CSS payloads to highlight editable DOM elements on hover, and maps clicks to floating UI modals.

  • Super Storage Fallback: It utilizes IndexedDB as a "super storage" fallback, gracefully handling massive, image-heavy draft saves that would normally crash standard 5MB browser localStorage limits.
  • Auto-Branching: When clients save their visual edits, the system automatically creates a specialized -client branch in the GitHub repository to strictly segregate their changes from the agency's master build, preventing accidental code destruction.
