Operator Help

Quick guide for employees to deploy samples, push revisions, manage staging branches, and handle access without breaking the workflow.

Deploy Sample

  1. Use Prompt Pack for client ZIP packs or Code Only for direct HTML paste.
  2. Set the target folder (example: eagle-eye).
  3. Click Preview to sanity-check before deploying.
  4. Click Deploy Base Build to push the root sample.

Deploy Revision

  1. Select a client directory first.
  2. Upload files (HTML/CSS/JS) or a ZIP containing the revision build.
  3. The system auto-calculates the next version (v1, v2, v3...).
  4. Push the branch, then copy the generated links for review/approval.

Staging Repo

  1. Click a client row to expand versions.
  2. Use Live, VS Code, Overwrite, Duplicate, or ZIP per version.
  3. New Branch creates the next revision from base or an existing version.
  4. Only delete when you are sure. Deletions can affect links employees or clients are using.

Access Control (Admins)

  • • Create employees in Security & Access.
  • • Open Manage Permissions and tick only the client folders they should see.
  • • Save access, then ask the employee to refresh / sign out and sign in again if permissions look stale.
  • • If permissions fail, check backend database records and Cloud Run logs before retrying.
