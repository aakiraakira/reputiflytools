Fulfillment SOP v4.2

Website Design Fulfillment SOP

Generating a website draft is only the first part of fulfillment. True fulfillment covers strict scope adherence, secure staging deployments, tracking revision feedback, and utilizing Agentic AI to automate code changes. Follow these steps exactly.

Phase 1: Creating the Draft

1

Context & Structure Prep

Before you prompt an AI, you must gather the exact constraints. We use two internal tools for this.

1. Get the Sitemap

Download the `sitemap.json` from Reputifly Scope. WARNING: This is a legally approved document. Do not deviate.

Scope View
Download the Sitemap from here →

2. Get the Brand Styles

Export the `brand-kit.json` from Reputifly Forge. This tells the AI exactly what hex colors and fonts to use.

Forge View
Get Brand Styles Here →

Your Required File Structure

Create a folder for your client project. Before generating code, your folder should look exactly like this:

📁 Client_Project/
  ├── 📄 sitemap.json // Downloaded from Scope
  ├── 📄 brand-kit.json // Downloaded from Forge
  └── 📁 assets/ // Images & PDFs (See Step 2)
2

Asset Management

Agentic AI (like Cursor or Claude) needs to know where to find images and PDFs. You have two options for handling assets.

A

Local Folder (Standard)

Place all client logos, background images, and PDFs into an `assets` folder inside your project directory.

📁 assets/
  ├── 🖼️ logo-transparent.png
  ├── 🖼️ hero-background.jpg
  └── 📄 company-profile.pdf

You must tell the AI where this folder is when prompting it in Step 3.

B

Cloud Method (Optional)

If you prefer using live URLs in your HTML, use our asset server. Copy the generated raw URLs to give to the AI.

Deploy Images Here →
3

AI Draft Generation

Feed your gathered JSON context and assets into your Agentic AI platform.

Base AI Prompt

System Instructions: You are generating the full HTML/Tailwind code for a website draft. 1. Adhere STRICTLY to the attached `sitemap.json`. You must generate exactly what is listed—nothing more, nothing less. Do not invent unapproved sections. 2. Utilize the attached `brand-kit.json` for all core aesthetics (Hex colors, typography, border-radius). However, you are permitted to intelligently optimize the theme colors for superior UI contrast and modern flow. 3. Assets: All relevant images and PDFs are located in the `/assets/` folder (or use these URLs: [Insert URLs if using Cloud Method]). Use them appropriately in the HTML. Ensure the output is fully responsive and complete.
4

Deploy & Tracking Hub

The sitemap is automatically injected from the JSON downloaded from Scope. More info can be found in the Help Guide in reputifly.org/foundry/deploy.

Deploy your Draft over here and send the client your link. After that, it is revisions already.

  • Zip your HTML files.
  • Upload them to Reputifly Foundry Deploy.
  • Copy the generated tracking link and send it to the client.

Phase 2: Revision & Management

5

The Client Revision Portal

The Client Revision Portal interface where clients point and click to leave feedback on live elements.

Get the `sitemap.json` from Scope and upload it inside the Revision Portal. Copy the generated link, send it to the client, and wait for them to make their changes.

Prepare Client Revision Link →
EXCEPTION: If the client is elderly or severely non-tech-savvy, they are permitted to provide manual revision lists via email or WhatsApp.
6

Executing AI Revisions

When a client submits feedback, you will receive an email notification. However, the entire workflow is now managed centrally via the Revision Dashboard.

  1. Log into the Revision Dashboard at reputifly.org/revision/.
  2. Locate the incoming client submission. You can download the Master .zip payload directly from here to feed into your AI agent.
  3. Click "Process in Tracker" to automatically launch the interactive visual checklist and sync live updates with the client.
  4. Toggle the Lock Edits switch when revisions are finalized to freeze the client's read-only view.

Access the central hub to manage all pending revisions:

Open the Revision Dashboard here →
7

Final Polish & Closure

Unlock the live visual editor from the Staging Repo menu to allow the client to edit text manually.

Unlocking Live Text Editing

If the client is satisfied with the design but wants to tweak text content endlessly, unlock the editor for them.

How to Unlock:

  1. Navigate to reputifly.org/foundry/deploy.
  2. Go to the Staging Repo.
  3. Search for the specific project.
  4. Click the 3 dots (⋮) menu.
  5. Select "Client unlock editor".
Open Foundry Deploy Console →

Project Closure: The fulfillment cycle ends when final approval is recorded in the Client Tracking dashboard.

Web Portals

  • Reputifly Forge

    Generates the stateless visual editor links for client feedback.
