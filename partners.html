System Architecture Overview

The Reputifly Revision-Fulfillment System is designed to solve the bottleneck of manual coding updates. Instead of interpreting client feedback and manually writing code, we leverage a structured Input-Processing-Output pipeline.

1. Input Aggregation

Clients provide feedback via a structured Google Doc. This document acts as the "Source of Truth," containing specific requests per page (e.g., "Change header color," "Update hero text"). We capture this raw data.

2. Intelligent Processing

We feed three critical components into Gemini:
1. The System Prompt (Rules of engagement),
2. The Client Doc (The requests), and
3. The Current Code.
The AI processes the request specifically for that page context.

3. Rapid Output

Gemini returns fully updated, production-ready HTML/CSS code. We then take this code and deploy it directly into WordPress using our specific "Zero Gap" configuration, ensuring a pixel-perfect update in minutes, not hours.

Fulfillment Protocol

This is the official workflow for processing client revisions. We use a standardized "3-File System" to eliminate errors and ensure quality.

The Golden Rule: One Page Per Prompt

NEVER attempt to fix multiple pages (e.g., Home AND Contact) in a single prompt.

Why? AI context windows are finite. If you feed too much code, it will "hallucinate" and delete parts of the layout. Always process one page at a time.

Mobile & Quality Assurance

Warning: Over 50% of traffic is mobile. A broken mobile UI means the job is REJECTED.

Rule 1: The "Horizontal Scroll" Killer

If a user can scroll left or right on their phone, the page is broken. This usually happens when an element is wider than the screen (e.g., fixed width images).

‚ùå BAD UI Page wiggles side-to-side. Content is cut off.
‚úÖ GOOD UI Page scrolls ONLY up and down. All content fits width.

Rule 2: The Strict "No Emoji" Policy

Text emojis (üöÄ, ‚úÖ, üòä) look unprofessional and render differently on iPhone vs Android.

The Mandate
  • NEVER use emojis in headings, subtext, or paragraphs.
  • ALWAYS use SVG Icons (Heroicons, FontAwesome) if you need a visual element.

The "Mobile Fixer" Prompt

If you spot these issues, feed the code back to Gemini with this prompt:

I am pasting the current code. It has MOBILE UI ISSUES. 1. HORIZONTAL SCROLL: The page overflows horizontally on mobile. Ensure all containers have `max-w-[100vw]` and `overflow-x-hidden`. Fix any fixed-width elements. 2. NO EMOJIS: Replace all text-based emojis (e.g., üöÄ, ‚úÖ) with inline SVG icons (using heroicons or similar standard paths). 3. FONT SIZING: Ensure headings do not break the screen width on mobile (use `break-words` or responsive text sizes). 4. For Emojis in sub-text or headings (filler text content in the website eg, review section, description, sub heading), just completely remove it.
